# Portfolio Dashboard

Private client-side retirement portfolio dashboard and Monte Carlo PWA.

The active application project is [`MVP-Portfolio-Dash-3A-Validated/`](MVP-Portfolio-Dash-3A-Validated/). Its authoritative documents are:

- [Requirements v2.3](MVP-Portfolio-Dash-3A-Validated/requirements.md)
- [Roadmap v2.3](MVP-Portfolio-Dash-3A-Validated/roadmap.md)
- [Project and validation guide](MVP-Portfolio-Dash-3A-Validated/README.md)
- [Changelog](MVP-Portfolio-Dash-3A-Validated/CHANGELOG.md)
- [Version 2.3 baseline validation](MVP-Portfolio-Dash-3A-Validated/docs/baseline_validation.md)

The accepted baseline includes Phases 0 through 3A plus the Version 2.3 predefined/editable Finnhub-key migration. Phase 1D remains intentionally deferred until validation can occur on the target home Wi-Fi.

Serve the application from its project directory:

```bash
cd MVP-Portfolio-Dash-3A-Validated
python3 -m http.server 8000
```

Then open `http://localhost:8000/` or `http://localhost:8000/tests/index.html` in Firefox.

The configured repository remote is `git@github.com:KCs-IRA-Dash-Monte-Carlo/Portfolio-Dashboard.git`.
