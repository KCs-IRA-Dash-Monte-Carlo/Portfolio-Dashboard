Build Phase 3A Portfolio Engine.

Authoritative spec: requirements.md, Version 2.2.
Do not build benchmark analytics, charts, Monte Carlo, or the holdings/lots UI yet.
Do not call Finnhub /stock/candle.
Use the existing Historical Data Service for all historical portfolio series.

Create:
- js/portfolio/portfolio-model.js
- js/portfolio/lot-model.js
- js/portfolio/portfolio-validation.js
- js/portfolio/portfolio-engine.js
- js/utils/number-utils.js
- js/utils/date-utils.js
- tests/calculation-tests.html updates

Requirements:
1. Support holdings.
2. Support multiple acquisition lots per holding.
3. Each lot stores ticker, shares, purchase price per share, acquisition date, and audit note.
4. Adding to an existing holding creates a new lot unless explicitly editing an existing lot.
5. Support fractional shares.
6. Use JavaScript Number precision internally; round only for display.
7. Validate invalid tickers, duplicate handling, invalid dates, future dates, zero/negative shares, invalid prices, zero/negative prices, NaN, and Infinity.
8. Duplicate ticker lots are valid under one holding.
9. A ticker may exist as both a holding and benchmark only if records remain distinct.
10. Provide stable public calculations for lot cost, holding cost basis, portfolio cost basis, position value, portfolio value, market-value weights, and portfolio inception date.
11. Use current Finnhub quote snapshots only for current position and portfolio values.
12. Use HistoricalDataService.getAlignedSeries() for historical account-value and portfolio performance series. Do not read Stooq source files or IndexedDB directly from the Portfolio Engine.
13. Model fixed-share buy-and-hold behavior with no periodic rebalancing. Historical market-value weights must float from shares and prices without implying daily rebalancing.
14. Respect lot acquisition dates. Portfolio inception is the earliest active acquisition date unless an explicit inception date is supplied.
15. Treat new lots as external flows when chaining historical portfolio returns so contributions do not appear as investment return.
16. Label every Stooq-derived output as a split-adjusted, dividend-unadjusted price-return approximation.
17. Do not label historical output as exact total return, dividend-reinvested total return, brokerage-reconciled performance, or tax-adjusted performance.
18. Surface missing, stale, insufficient, and quality-warning states received from the Historical Data Service. Do not fabricate missing history.
19. Provide deterministic calculation tests for cost basis, position value, portfolio value, market-value weights, acquisition-date lot activation, contribution-neutral return chaining, and Version 2.2 labels.
20. Provide exact file contents and manual tests.
